Detailed Description

The Type class is a simplified representation.

There is no derived types or composite types. All types are primitive and are represented by this class through its tag member. The reasons why Type is still implemented as a class (not directly a tag in the Value) are:

  1. Type has it own methods.
  2. make interfaces of both Type and Value cleaner.
  3. stand more aligned with llvm.
  4. future extension makes easier.
